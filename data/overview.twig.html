Welcome to the Open Collections API Documentation ! Below you'll find tutorials that will teach you how to use the Open Collections API and reference documentation for all its moving parts.

Where To Go

  • If you haven't used an API before, head on over to API Basics which will get you started on accessing our data.
  • If you have already used APIs and you just want to know how to get at our data, skip the basics and head straight on over to our Requests section and take a look a our Reference section. (You'll need an API key first.)
  • Once you've made a request, you might need some help making sense of what it all means. In responses, we explore the structure of objects and the fields within those objects that the API returns.
  • Now you should be ready to start building something with our API, however you might like to know the small print, so go check out our API's policies.
  • Visit and read our terms of use page, before using the API.
  • Reach us at library.opencollections@ubc.ca for inquiries. (or if you make something cool you'd like us to feature)

The Essentials

If you need to start straight away, here is how:

  • {{data.api_url}} is the base URL of the Open Collections API.
  • In order to pass requests to the API, you'll need to register an API key first. You also need to append &api_key=[:YOUR_API_KEY] (where [:YOUR_API_KEY] is the API key you received after registering for the Open Collections API Key) to any requests you make.
  • All records are returned by default as JSON-LD objects, however you can request other formats as described over at output formats
  • We've got lots of collections & fields you can query, and a whole lot of ways you can query them.
  • The main resources the API returns are Collection Metadata and Item Metadata, for Image Metadata or image downloads you will need to check out our IIIF documentation (Media API for audio & video coming soon)
